# 금시세 일일 업데이트 서비스 — PRD

## 1. 제품 개요

### 1.1 배경
국내 실물 금(3.75g·1돈 기준) 고시가는 하루에도 여러 차례 바뀔 수 있으며, 사용자는 최신 고시 내역과 날짜별 추이를 한 화면에서 빠르게 확인하고 싶어합니다. 운영자는 외부 소스에서 데이터를 안정적으로 수집해 저장하고, 서비스 화면은 저장된 데이터만 기준으로 일관성 있게 보여줄 필요가 있습니다.

### 1.2 제품 목표

- 외부 금시세 API 데이터를 수집해 내부 저장소(Supabase)에 일관되게 적재한다.
- 저장된 데이터 기반으로 차트와 표를 제공해 조회 속도와 안정성을 높인다.
- 로컬 실행과 Vercel 배포가 모두 가능한 형태로 운영한다.

### 1.3 현재 배포 상태

- 데이터 저장소: Supabase `public.gold_quotes`
- 웹 백엔드: FastAPI
- 배포 플랫폼: Vercel Python Functions
- 운영 URL: `https://goldprice-phi.vercel.app`

## 4. 기능 요구사항

### 4.1 데이터 수집

- 소스: 한국금거래소 공개 API `POST /api/price/chart/list`
- 수집 스크립트: `scripts/sync_to_supabase.py`
- 저장 대상: Supabase `public.gold_quotes`
- 중복 처리: `announced_at` 기준 upsert
- 동일 시각 중복 응답이 있을 경우 마지막 데이터를 유지

### 4.2 데이터 저장

- 저장 필드
  - `announced_at`
  - `s_pure`
  - `p_pure`
  - `p_18k`
  - `p_14k`
  - `raw`
- 공개 조회는 가능하되 쓰기는 `service_role` 기반 서버 작업으로 제한

### 4.3 웹 조회

- API: `GET /api/prices`
- 조회 기준: 외부 API가 아니라 Supabase 저장본
- 입력 파라미터
  - `days`
  - `table_limit`
- 응답 데이터
  - `meta`
  - `daily`
  - `table`

### 4.4 화면

- 기간 선택
  - 최근 90일
  - 최근 180일
  - 최근 1년
  - 최근 2년
- 라인 차트
  - 일자별 마지막 고시 시세 기준
  - 항목별 표시 토글
- 표
  - 최신순 페이지네이션
  - 헤더 흰색 배경, 검정 글자
  - 값은 흰색 텍스트
  - 각 가격 열별 최고가 빨간색, 최저가 파란색 강조

### 4.5 새로고침 동작

- 현재 `새로고침` 버튼은 외부 API를 직접 재수집하지 않음
- 버튼 클릭 시 Supabase 저장 데이터를 다시 읽어 화면만 갱신
- 외부 동기화는 별도 스크립트 실행으로 수행

### 4.6 엑셀 저장

- 스크립트: `scripts/fetch_gold_to_excel.py`
- 기본 출력: 최근 100건 `.xlsx`
- 목적: 운영 확인, 수기 전달, 백업

## 5. 비기능 요구사항

- 응답 속도: 일반 조회에서 수 초 내 응답
- 안정성: 화면은 외부 사이트 직접 호출이 아닌 Supabase 저장본을 조회
- 배포성: 로컬 uvicorn 실행과 Vercel 배포 둘 다 가능해야 함
- 보안: `.env`, service role 키, 출력 엑셀은 저장소에 커밋하지 않음
- 가시성: 출처와 참고용 고지를 화면에 표시

## 7. 시스템 아키텍처

### 7.1 수집 흐름

1. 운영자가 `scripts/sync_to_supabase.py` 실행
2. 외부 API에서 최근 N일 데이터를 조회
3. 정규화 후 중복 시각 제거
4. Supabase `gold_quotes`에 upsert

### 7.2 조회 흐름

1. 사용자가 웹 페이지 접속
2. 프론트가 `/api/prices` 호출
3. FastAPI가 Supabase `gold_quotes` 조회
4. 서버가 `daily` 집계와 `table` 데이터를 생성
5. 프론트가 그래프와 표를 렌더링

### 7.3 배포 구조

- `web/main.py`: FastAPI 앱
- `api/index.py`: Vercel 엔트리포인트
- `vercel.json`: 모든 요청을 Python 함수로 라우팅

## 8. 운영 및 배포

### 8.1 로컬 실행

```bash
uvicorn web.main:app --host 127.0.0.1 --port 8765
```

### 8.2 배포 환경

- 플랫폼: Vercel
- Framework Preset: `Other`
- Root Directory: 저장소 루트
- 환경 변수
  - `SUPABASE_URL`
  - `SUPABASE_SERVICE_ROLE_KEY`

### 8.3 운영 시 주의사항

- Vercel 프로젝트가 `Next.js`로 잘못 생성되면 Python 앱 배포가 실패함
- 이 경우 프로젝트를 `Other` preset으로 다시 생성해야 함
- `SUPABASE_SERVICE_ROLE_KEY`에는 `service_role`만 사용 가능
- `sbp_...` 토큰은 사용 불가

## 10. 리스크

- 외부 API URL 또는 응답 스키마 변경 시 수집 실패 가능
- service role 키 노출 시 보안 위험
- 외부 데이터와 실제 거래 가격 차이 가능
- 현재 새로고침 버튼이 외부 동기화까지 수행하지 않음

대응:

- 수집 스크립트 실패 로그 확인
- 비밀값은 `.env`로만 관리
- 출처 및 참고용 문구 유지
- 향후 동기화 버튼 또는 스케줄러 추가 검토
